# タダサポ管理システム 操作マニュアル v1.12.8

**作成日:** 2026/01/27
**更新日:** 2026/06/11
**対象:** スタッフ / システム管理者

---

## v1.12.8 変更点

- 管理者向け: 設定管理の「その他」タブに **「案件キーをcase_id経由で結合（S1 Stage4）」** トグルを追加しました。ONにすると案件と対応記録の結合を不変IDで行い、日付キーの表記ブレを吸収します（通常はONのまま運用。問題時はOFFで即時に従来挙動へ戻せます）。一般利用者の操作・画面に変更はありません。

## v1.12.7 変更点

- 内部改善（操作手順の変更なし）: 案件の管理に使う内部キーを安定化する基盤を追加しました（「完了しても未対応に戻る」事象の根治に向けた段階対応）。通常利用への影響はありません。
- 管理者向け: 設定管理ダイアログに「メンテナンス：案件キーマップ Backfill」を追加しました（①診断 ②ドライラン ③本実行）。既存案件に内部IDを一括付与するための管理機能で、診断・ドライランは読み取り専用、本実行のみデータに追記します（再実行しても重複しません）。

## v1.12.6 変更点

- 管理モードで案件を担当者に振り分けた後に**完了しても、画面が「未対応・担当者未設定」に戻ってしまう**不具合を修正しました。操作手順の変更はありません（完了報告は従来どおりで、正しく「完了」タブに担当者付きで表示されます）。

## v1.12.5 変更点

- 管理モードで案件詳細の**担当者バッジから担当者を変更**する操作が、エラーになって反映されない不具合を修正しました。操作手順の変更はありません（変更後はトースト表示とともに即時反映されます）。

## v1.12.4 変更点

- 管理者が案件詳細の「**今年度利用数**」（n / 上限）の表示をクリックし、**実際の利用回数を手動で修正**できるようになりました。
  - 例: システム外（電話・紙・対面など）で受けた依頼を年度カウントに足す／重複・誤カウントを減らす。
  - 入力は「**目的の回数をそのまま入力**」する方式です（例: 5 と入力すれば今年度利用数が 5 になります）。
  - 修正は「**同じメールアドレス＋同じ年度**」の全案件に反映されます（利用回数は事業所メール単位で年度集計されるため）。
  - 同じポップアップで従来どおり「最大利用回数（上限）」も変更できます。
  - すべての修正は監査ログに記録されます。

## v1.12.3 変更点

- 管理者が「新規案件」から手動追加した案件も、Googleフォームでの申込と**同じメールアドレス**であれば、年度の利用回数（年間上限）に合算してカウントするようになりました（メールの大文字・小文字や前後の空白の違いは無視して同一人物と判定します）。
- 手動追加した直後に受付日が「manual_…」と表示される不具合を修正しました。

## v1.12.2 変更点

- 送信メールの差出人名（例:「NPO法人タダカヨ」）が受信側で文字化けする不具合を修正しました。
- 日程確定カレンダーで、選択中の枠が「✓ 選択中」の緑バーとして常に表示されるようになりました。
- 日程確定カレンダーで、前後のバッファ時間や既存予定に枠を重ねて選択できないようになりました（ダブルブッキング防止）。

## v1.12.1 変更点

- メール予約送信を廃止しました。メールは即時送信、後で送る文面は下書き保存を使用してください。

## v1.12.0 新機能ハイライト

### 日程確定モーダルが大幅刷新されました

「日程の確定・変更」ボタンを押すと、画面に**空き状況カレンダー**が表示されます。

- 📅 **週ビュー / 月ビュー切替**: チームタダカヨ + タダスクの予定を一目で確認
- 🖱 **ドラッグで時間枠選択**: 空いている時間をドラッグすると、上の日時欄が自動入力されます
- 🟡 黄色 = チームカレンダー / 🔴 赤 = タダスク等の表示専用カレンダー
- 斜線部分 = 既存予定の前後30分のバッファ（重複判定の対象）

### 方法=Zoom の挙動変更（重要）

- **チームタダカヨカレンダーへの登録は必須**になりました（重複防止のため）
- 「個人/共有カレンダーにも登録する」のチェックボックスは個人カレンダーへの追加登録のみを制御
- **Zoom 利用時のみ重複チェックが発動**します。重複検知時は送信できません（赤帯警告）
- GoogleMeet / 電話 / 対面 / メール等の方法では重複チェックは行いません

### 「いつものタダスクID」モード

設定シートに `ZOOM_FIXED_URL` / `ZOOM_FIXED_ID` / `ZOOM_FIXED_PASS` を入力しておくと、方法=Zoom 選択時に「新規発行 / いつものタダスクID」のラジオが表示されます。
固定IDモードでは Zoom API を呼ばず、登録済み URL を再利用します（タダスク業務など、同じ Zoom を繰り返し使う場合に便利）。

## 1. 前提条件

- Google アカウント（tadakayo.jp ドメイン）を保有していること
- 「タダメンマスタ」シートのC列に自分のメールアドレスが登録されていること
- 管理者操作には「タダメンマスタ」D列のロールが `admin` に設定されていること

---

## 2. 画面構成と表示モード

### 2-1. 表示モードの切替

画面上部に **通常 / 閲覧 / 管理** の3つのモードボタンがあります。

| モード | 表示範囲 | 操作 | 表示条件 |
|:---|:---|:---|:---|
| **通常** | 自分が担当する案件（未対応は全員共有） | 全操作可能 | 全スタッフ |
| **閲覧** | 全案件 | 読み取りのみ | 全スタッフ |
| **管理** | 全案件 | 全操作 + 管理者専用操作 | 管理者のみ |

### 2-2. 新着バッジ

各ステータスタブの右側に **`+N`** の赤いバッジが表示されます。「最後にそのタブをクリックした時点から増えた案件数」を示します。

- タブをクリックすると既読化されバッジが消えます
- 管理モードでは、ヘッダーの **Bell アイコン** でバッジのON/OFFを切り替えられます
- 閲覧モードではバッジは表示されません

## 3. 案件対応の基本フロー

### 3-1. 未対応案件を担当する

1. **「未対応」タブ** で担当したい案件を選択します
2. 年間利用回数が上限内の場合:
   - **「メール送信して担当」** — メール作成モーダルが表示されます。内容を確認・編集して送信
   - **「担当する（メールなし）」** — メール送信なしでアサインされます
3. 年間利用回数が上限に達している場合:
   - **「回数超過」** — 回数超過メールを送信し、案件を「対応不可」に変更します

### 3-2. 日程を確定する

1. 担当中の案件を選択
2. **「日程・方法を入力」** ボタンをクリック
3. 実施日時・対応方法（Google Meet / Zoom / 対面等）を入力して保存
4. Google カレンダーに予定が自動作成され、Google Meet URL が発行されます（Zoom 選択時は Zoom ミーティングが作成されます）
5. 日程確定後、確定メールの作成モーダルが自動表示されます

### 3-3. 完了報告を行う

1. 担当中の案件を選択
2. **「実施記録・完了報告」** ボタンをクリック
3. 実施内容・備考・対応ツール・サービス種別・都道府県を入力
4. ファイルを添付する場合は「ファイルを追加」または画像をドラッグ&ドロップ（最大5件）
5. **「完了にする」** または **「保存する（対応中に戻す）」** を選択して保存

### 3-4. 案件を再開する（継続支援）

完了案件に対して追加サポートを行う場合:

1. **「完了」タブ** で対象案件を選択
2. **「再開する」** ボタンをクリック
3. 対応回数が増え（2回目、3回目）、「対応中」に戻ります

再開の条件:
- 対応回数が上限（デフォルト3回）未満であること
- 今年度の利用回数が年間上限（デフォルト10回）未満であること

### 3-5. 案件をキャンセルする

1. 担当中の案件を選択
2. **「キャンセル」** ボタンをクリック
3. キャンセル理由を入力（1回目のみ）
4. 確認後キャンセルされます（担当者・サブ担当・管理者が実行可能）

## 4. メール機能

### 4-1. メールの送信モード

| モード | 用途 |
|:---|:---|
| **初回メール** | 担当アサイン時の最初のメール（テンプレートあり） |
| **新規メール** | 新しいスレッドを立てて送信 |
| **返信** | 既存のスレッドに返信 |
| **日程確定メール** | 日程確定時の通知メール（テンプレートあり） |
| **回数超過メール** | 年間上限超過時の断りメール（テンプレートあり） |

### 4-2. CC / BCC の設定

- メール作成モーダルの「CC/BCCを追加」から任意のアドレスを追加できます
- 管理者が設定した **自動CC（MAIL_FORCE_CC）** は全メールに自動付与されます（メール画面で確認可能）
- サブ担当が設定されている場合、メイン担当⇔サブ担当は自動的に CC に追加されます

### 4-3. メール下書きの保存と復元（v1.11.0）

- メール作成モーダルで **「下書き保存」** ボタンをクリックすると下書きが保存されます
- 次回同じモーダルを開いた時に「下書きを復元しますか？」と確認が表示されます
- 送信が完了すると下書きは自動的に削除されます
- 保存した下書きがある案件には **「下書きあり」** バッジが表示されます

### 4-4. メールの予約送信（v1.12.1で廃止）

- 予約送信は、送信者がトリガー作成者アカウントになる可能性があるため廃止しました
- メールは **即時送信** してください
- 後で送る文面を残す場合は **「下書き保存」** を使用してください
- 既存の未送信予約は管理者が無効化し、自動送信されません

## 5. 検索・フィルタ

### 5-1. キーワード検索

サイドバー上部の検索欄で**常時**検索できます（事業所名・担当者名・メール・内容・種別・都道府県の横断検索）。

### 5-2. 詳細フィルタ（フィルタアイコンをクリックで展開）

| フィルタ | 内容 |
|:---|:---|
| **期間プリセット** | 今月 / 先月 / 今年度 / 前年度 をワンクリックで設定 |
| **カスタム期間** | 開始日〜終了日を直接入力 |
| **都道府県** | 絞り込み |
| **サービス種別** | 絞り込み |
| **対応ツール** | 絞り込み |
| **並び順** | 最新順 / 古い順 |
| **担当者フィルタ** | 全件 / 未割当 / 個別担当者（管理モードのみ） |

適用中のフィルタは**チップ（pill）**として表示され、個別に削除できます。

## 6. サブ担当（OJT）機能

- 案件詳細の「サブ担当を追加」から1名を指定できます（管理者・担当者が操作可）
- サブ担当はメール送信・記録修正・キャンセル等の操作が可能です
- サブ担当とメイン担当は互いにメール送信時に CC に自動追加されます

## 7. 管理モード操作（管理者のみ）

### 7-1. インライン編集

管理モードでは、案件詳細画面の以下の要素を**クリックするだけ**で直接編集できます。

| クリック箇所 | 操作内容 |
|:---|:---|
| ステータスバッジ（例：「対応中」） | ステータスをドロップダウンから選択して変更 |
| 担当者バッジ（例：「田中花子」） | 担当者を検索絞り込み付きドロップダウンから変更 |
| 案件回数（例：「2 / 3」） | 案件ごとの対応上限を特例変更（空欄で全体設定に戻す） |
| 今年度利用数（例：「5 / 10」） | 年間上限を特例変更（空欄で全体設定に戻す） |

ポップオーバーは **Escape キー** または外側クリックで閉じられます。

### 7-2. 権限管理ダイアログ

ヘッダーの「権限管理」ボタンから開きます。

- スタッフの名前・メールで検索できます（100名以上対応）
- ロールを「管理者 / タダメン / 無効」に変更できます
- 自分自身のロール変更は禁止されています
- 新規メンバーの追加はスプレッドシートのタダメンマスタで行います（アプリからは不可）

### 7-3. 設定管理ダイアログ

ヘッダーの「設定管理」ボタンから開きます。カテゴリ別に設定を管理できます。

| カテゴリ | 主な設定 |
|:---|:---|
| **上限設定** | 年間利用上限・案件回数上限 |
| **メール設定** | 各種メールテンプレート・自動CC |
| **連携設定** | 共有カレンダーID・添付フォルダ・Zoom API |
| **その他** | 対応ツール一覧・ツール月間上限 |

### 7-4. データ抽出

管理モード → 「データ抽出」ボタンから、案件データを CSV 形式でエクスポートできます（都道府県・事業所名・項目選択UI）。

## 8. トラブルシューティング

| 症状 | 確認事項 |
|:---|:---|
| 「アクセス権限がありません」が表示される | アクセスしているGoogleアカウントのメールアドレスが「タダメンマスタ」C列に登録されているか確認 |
| 自分の案件が見つからない | 表示モードが「通常」になっているか確認。管理者に担当者設定を依頼 |
| 編集ボタンが押せない・グレーアウトしている | 「閲覧」モードになっていないか確認。「通常」モードに切り替える |
| 新着バッジが消えない | 該当タブをクリックすると既読化される。管理モードの Bell ボタンでバッジをOFFにすることも可能 |
| 「管理」ボタンが表示されない | タダメンマスタのD列（ロール）に `admin` が設定されているか確認 |
| メールが送信されない | MAIL_DRY_RUN が `true` になっていないか設定シートを確認 |
| 予約メールが送信されない | 予約送信は v1.12.1 で廃止済み。下書き保存または即時送信を使用 |
| カレンダーに予定が作成されない | Calendar Advanced Service が有効化されているか確認。SHARED_CALENDAR_ID の設定を確認 |
