④ 移行の考え方（既存ハードルールと両立）

  • 案件リストはIMPORTRANGEで書込不可 → 列追加せず 案件キーマップ（新規シート）で 自然キー→case_id を管理。
  • URL不変・固定deploymentId → 配信構成は不変。突合は canonicalCaseKey_() に統一。
  • 段階導入（各段階でE2E回帰必須）: ①正準化＋孤立検出で止血 → ②案件キーマップ導入＋FKバックフィル → ③FKをcase_idへ切替。
