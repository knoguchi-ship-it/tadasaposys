# 作業範囲記述書 (SOW) - タダサポ管理アプリケーション構築・導入

**Project Name:** タダサポ管理アプリケーション (v1.3 Fixed Spec)  
**Date:** 2026/01/27

## 1. プロジェクトの目的
Googleスプレッドシート上で分散管理されていたタダサポ業務を一元化し、定義された仕様（Spec-Driven Development）に基づき、堅牢かつ効率的な管理システムを構築する。

## 2. 作業範囲 (Scope of Work)

### 2.1 仕様検証とDB整備 (Spec Validation)
*   **データモデル整合性の確認**: スプレッドシートの構造が、SDDの 「1. データモデル仕様」 に完全に準拠していることを確認する。
    *   特に「案件リスト」のI列（9列目）が「サービス種別」であることを保証する。
*   **マスタ整備**: 運用開始前に「タダメンマスタ」へ全スタッフのメールアドレスを登録する。

### 2.2 実装作業 (Implementation)
*   **バックエンド**: `Code.gs` (v1.3) をデプロイする。SDDの 「2. インターフェース仕様」 を満たすAPIエンドポイントを提供する。
*   **API連携**: Google Cloud Project (GCP) 連携設定を行い、Google Calendar API および Gmail API を有効化する。
*   **フロントエンド**: `index.html` (v1.3) を配置し、SDD SEC-01（エラー画面）を含むUIを構築する。

### 2.3 テスト (Verification)
SDDに定義された仕様に基づき、以下のテストを実施する。
1.  **認証テスト (SEC-01)**: マスタ未登録ユーザーがアクセスした際、明確なエラー画面が表示されるか。
2.  **データ結合テスト (F-01)**: 案件リストのデータに、サポート記録のステータスが正しく結合されて表示されるか。
3.  **カレンダー同期テスト (Spec-CAL)**: 日時保存時にGoogleカレンダーへ予定が作成され、Meet URLが取得できるか。

## 3. 成果物
1.  **Code.gs (v1.3)**: バックエンドロジック
2.  **index.html (v1.3)**: フロントエンドアプリケーション
3.  **SDD (v1.3 Spec-Driven Document)**: 仕様書
4.  **SOW (Spec-Aligned Version)**: 本ドキュメント
