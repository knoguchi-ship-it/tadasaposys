⚙️ 設定ファイル

playwright.config.ts

Playwright 1.52.0 の推奨設定をベースに GAS + React SPA の特性に合わせて調整しています。

playwright.config.ts
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir:       './tests/e2e',
  fullyParallel: false,  // ローカルサーバー共有のため順次実行
  forbidOnly:    !!process.env.CI,
  retries:       process.env.CI ? 2 : 0,
  workers:       1,

  reporter: [
    ['html', { outputFolder: 'playwright-report', open: 'never' }],
    ['list'],
  ],

  use: {
    baseURL:    'http://localhost:3000',
    locale:     'ja-JP',
    timezoneId: 'Asia/Tokyo',
    trace:      'on-first-retry',     // 失敗時のみトレース収集
    screenshot: 'only-on-failure',  // 失敗時のみスクリーンショット
    video:      'retain-on-failure', // 失敗時のみ動画保存
    actionTimeout:     15_000,  // Babel 初期化に余裕を持たせる
    navigationTimeout: 30_000,
  },

  projects: [{
    name: 'chromium',           // GAS は Chrome 前提のため Chromium のみ
    use:  { ...devices['Desktop Chrome'] },
  }],

  webServer: {
    command:             'npx serve -s . -l 3000',  // テスト前に自動起動
    url:                 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,  // ローカルでは既存サーバー再利用
    timeout:             30_000,
    stdout: 'ignore',
    stderr: 'pipe',
  },
});

アーキテクチャ選定理由

✅
Page Object Model (POM) を採用した理由 UI の変更（セレクターの変更等）があっても、Page Object だけ修正すればすべてのテストに反映される。テストコードとロケーターを分離することで保守性が向上する。
✅
test.extend() Fixture を採用した理由 各テストで「アプリを開いてデータ読み込み完了まで待機」という共通セットアップが必要。Fixture として切り出すことでDRYになり、setup/teardown が自動管理される。

📄 Page Object Model

tests/pages/app.page.ts がアプリ全体の UI 操作を隠蔽します。

⚠️ GAS 特有のロケーター上の注意点

注意 タブボタンは role="tab"
ステータスタブ（未対応・対応中・完了等）は HTML で role="tab" が付与されています。

❌ 間違い: 
tabList.getByRole('button', { name: '未対応' })
✅ 正しい: 
tabList.getByRole('tab', { name: '未対応' })
注意 アクションボタンは fixed overlay
「担当する」「回数超過」「N回目を開始」ボタンは fixed bottom-6 のフローティングバーにあり、.max-w-3xl（詳細パネル）の外側にあります。

❌ 間違い: 
page.locator('.max-w-3xl').getByRole('button', { name: '回数超過' })
✅ 正しい: 
page.locator('[class*="fixed"][class*="bottom-6"]')
  .getByRole('button', { name: '回数超過' })

主要メソッド一覧

tests/pages/app.page.ts — 抜粋
export class AppPage {
  // アプリを開いてデータ読み込み完了まで待機
  async goto(): Promise<void> { ... }

  // タブをクリック (role="tab" を使用)
  async clickTab(tab: TabKey): Promise<void> {
    await this.tabList.getByRole('tab', { name: tab }).click();
  }

  // 表示モードを切替 (aria-pressed で確認)
  async switchMode(mode: DisplayMode): Promise<void> {
    const btn = this.modeGroup.getByRole('button', { name: mode });
    await btn.click();
    await expect(btn).toHaveAttribute('aria-pressed', 'true');
  }

  // ケースカードを #case-list-panel 内で取得
  getCaseCards(): Locator {
    return this.page.locator('#case-list-panel button')
      .filter({ has: this.page.locator('h4') });
  }

  // アクションボタンを fixed overlay 内で取得
  getActionButton(label: string | RegExp): Locator {
    return this.actionBar.getByRole('button', { name: label });
  }
}

🔧 Fixture パターン

test.extend() を使ってカスタム Fixture を定義します。各テストに「goto 済みの AppPage」が自動で提供されます。

tests/fixtures/index.ts
import { test as base } from '@playwright/test';
import { AppPage } from '../pages/app.page';

export const test = base.extend<{ appPage: AppPage }>({
  appPage: async ({ page }, use) => {
    const appPage = new AppPage(page);
    await appPage.goto();   // ① アプリを開く
    await use(appPage);     // ② テスト実行
    // teardown は Playwright が自動管理
  },
});

export { expect } from '@playwright/test';

waitForLoaded の実装

⚠️
waitForLoadState('networkidle') は使用しない esm.sh・unpkg・Google Fonts 等の CDN が HTTP/2 の接続を維持するため、networkidle が永遠に待機状態になる場合があります。
正しい waitForLoaded の実装
async waitForLoaded(): Promise<void> {
  // タブリスト div が表示 = React 初期化完了
  await this.tabList.waitFor({ state: 'visible', timeout: 60_000 });

  // タブ要素(role="tab")が表示 = モックデータ(300ms delay)ロード完了
  await this.tabList.getByRole('tab').first()
    .waitFor({ state: 'visible', timeout: 15_000 });
}

🔧 発見した問題と解決

✅ 解決済み Issue #1: タブボタンの ARIA ロールが button ではなく tab

症状: tabList.getByRole('button', { name: '未対応' }).click() が 15秒タイムアウト

原因: HTML の <button role="tab"> により ARIA ロールが tab に上書きされていた

確認方法:

# HTML で確認
grep -n 'role="tab"' index.html
# → <button key={s} role="tab" aria-selected=...

解決: getByRole('button') → getByRole('tab') に変更

✅ 解決済み Issue #2: waitForLoadState('networkidle') が永遠に待機

症状: fixture の setup が 15秒タイムアウトで失敗するテストが続出

原因: esm.sh・Google Fonts 等の CDN が HTTP/2 の Keep-Alive 接続を維持するため、networkidle（500ms 以上リクエストなし）の条件が満たされない

解決: waitForLoadState('networkidle') を削除し、代わりに UI 要素（role="tab"）の表示を待つ

✅ 解決済み Issue #3: アクションボタンが .max-w-3xl の外にある

症状: page.locator('.max-w-3xl').getByRole('button', { name: '担当する（メールなし）' }).isVisible() が常に false

原因: アクションボタン（担当する・回数超過・N回目を開始等）は fixed bottom-6 の フローティングオーバーレイにあり、詳細パネル（.max-w-3xl）の外側にある

解決: ロケーターを [class*="fixed"][class*="bottom-6"] 内に変更

✅ 解決済み Issue #4: 「再開する」ボタンが見つからない

症状: getActionButton('再開する') でボタンが見つからない

原因: 実際のボタンテキストは動的で "{supportCount+1}回目を開始"（例：「3回目を開始」）

確認方法:

grep -n "回目を開始" index.html
# → {(selectedCase.supportCount || 1) + 1}回目を開始

解決: 正規表現 /\d+回目を開始/ を使用

getActionButton(/\d+回目を開始/)
未解決（既知） Issue #5: WCAG color-contrast 違反

内容: Tailwind の薄色テキスト（text-slate-300・text-slate-400 等）が白背景との対比不足で WCAG 2.1 AA の基準（4.5:1）を満たしていない

影響箇所: 新着バッジ・日付表示・プレースホルダー等

現在の対応: テストで .disableRules(['color-contrast']) として除外。UI デザインの改善は別 Issue で管理

♿ アクセシビリティテスト

axe-core（@axe-core/playwright）を使用して WCAG 2.1 AA の自動チェックを実施します。

ℹ️
自動テストの限界 axe-core による自動チェックは WCAG 違反の約 57% のみ検出できます。スクリーンリーダーでの操作確認・キーボードナビゲーション等は手動テストが必要です。

テストコード例

07-a11y.spec.ts — axe 実行ヘルパー
import AxeBuilder from '@axe-core/playwright';

async function checkA11y(page) {
  const results = await new AxeBuilder({ page })
    .withTags(['wcag2a', 'wcag2aa', 'wcag21aa'])
    .exclude('[aria-hidden="true"]')
    .disableRules(['color-contrast'])  // 既知課題として除外
    .analyze();

  const critical = results.violations.filter(
    v => v.impact === 'critical' || v.impact === 'serious'
  );
  expect.soft(critical.length, ...).toBe(0);
}

🔄 CI/CD 設定例（GitHub Actions）

ℹ️
本プロジェクトは現在 CI/CD 未設定ですが、以下の設定例を参考に追加できます。workflows/ ディレクトリに playwright.yml として配置してください。
.github/workflows/playwright.yml
name: Playwright E2E Tests

on:
  push:
    branches: [ master, main ]
  pull_request:
    branches: [ master, main ]

jobs:
  test:
    timeout-minutes: 20
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v4

    - uses: actions/setup-node@v4
      with:
        node-version: 22
        cache: npm

    - name: Install dependencies
      run: npm ci

    - name: Install Playwright browsers
      run: npx playwright install chromium --with-deps

    - name: Run Playwright tests
      run: npx playwright test
      env:
        CI: true        # retries: 2 が有効になる

    - uses: actions/upload-artifact@v4
      if: failure()    # 失敗時のみレポートをアップロード
      with:
        name: playwright-report
        path: playwright-report/
        retention-days: 30
