1 アクセシビリティツリーを理解しよう

Playwright MCP はスクリーンショット（画像）ではなく、アクセシビリティツリー（構造化テキスト）でページを理解します。これが最も重要な概念です。

ブラウザが描画する画面には、視覚的な見た目とは別に「アクセシビリティツリー」という構造化データが存在します。スクリーンリーダーなどの支援技術が利用するもので、Playwright MCP はこれを活用して高速かつ正確にページ状態を把握します。

スナップショット vs スクリーンショット

✔ スナップショット（推奨）

  • 構造化テキスト 2-5KB
  • ref番号で要素を正確に操作
  • コンテキスト消費が最小限
  • 要素の役割・名前・状態を取得

✘ スクリーンショット

  • 画像データ 500KB-2MB
  • 座標計算が必要で不正確
  • 大量のトークンを消費
  • 視覚確認にしか使えない

アクセシビリティツリーの実例

以下は、browser_snapshot が返すデータの一例です。

- heading "お問い合わせ" [ref=e35]
- textbox "メール" [ref=e38]: user@example.com
- button "送信" [ref=e42]
🔑
ref番号とは： アクセシビリティツリーの各要素には [ref=e42] のような固有番号が付与されます。Claude はこの番号を使って要素を正確に操作します。ユーザーがref番号を覚える必要はありません。Claude が自動的に判断して使用します。

2 3つのベストプラクティス Best Practice

BP1: 初回は明示的に伝える

セッション最初は「Playwright MCP で」と明示的に指示してください。2回目以降は自然な日本語でOKです。

✔ 初回の指示

「Playwright MCP で http://localhost:3000 を開いて」

Claude が確実にMCPツールを認識して使用します。

✔ 2回目以降

「検索欄に『訪問』と入力して」

すでにMCPを認識しているので自然言語だけでOKです。

BP2: スナップショット優先

スクリーンショットではなくスナップショット（アクセシビリティツリー）を優先しましょう。

✔ こうする

「画面の状態を確認して」

➔ browser_snapshot が使われる

✘ こうしない

「スクリーンショットを撮って」

➔ 大量トークン消費

BP3: 段階的に操作する

複雑な操作は一度に全てを指示せず、段階的に進めると安定します。

1

ページを開いて状態確認

まずページを開き、スナップショットで現在の状態を把握します。

2

操作を指示（クリック、入力など）

確認した状態をもとに、具体的な操作を指示します。

3

操作後にスナップショットで結果確認

操作が意図通りに完了したかスナップショットで確認します。

3 やりたいこと逆引き表 New

「何をしたいか」から「どう指示すればいいか」を逆引きできる表です。コピーしてそのまま使えます。

🌐 ページを開きたい
「http://localhost:3000 を開いて」
🖱 ボタンを押したい
「〇〇ボタンをクリックして」
✏ テキストを入力したい
「検索欄に『訪問』と入力して」
🔃 ドロップダウンを変更
「方法をGoogleMeetに変えて」
📱 画面サイズを変更
「画面を375x812にして」
⚠ エラーを確認
「コンソールエラーを確認して」
⏪ 前のページに戻る
「前のページに戻って」
⏳ 待機する
「ローディングが消えるまで待って」
🗨 ダイアログに応答
「確認ダイアログでOKを押して」
✖ ブラウザを閉じる
「ブラウザを閉じて」
💡
ポイント： 特別なコマンドや構文を覚える必要はありません。日常の日本語で指示するだけで、Claude が適切なMCPツールを選択して実行します。

4 よくある間違い（Do / Don't） 重要

初めて使うときに陥りがちな間違いと正しいやり方をまとめました。

間違い 1：一度に大量の指示を出す

✔ こうする

1つずつ段階的に指示する

「まずページを開いて」➔「状態を確認して」➔「ボタンを押して」

✘ こうしない

「5つのテストを全部一気にやって」

途中でエラーが起きても原因の特定が困難になります。

間違い 2：状態を確認せずに操作する

✔ こうする

操作前にスナップショットで状態確認

「画面の状態を確認して」➔ 状態把握 ➔「〇〇をクリック」

✘ こうしない

状態を確認せずにいきなり操作

ページの読み込みが終わっていない等で操作が失敗することがあります。

間違い 3：毎回スクリーンショットを要求する

✔ こうする

「画面の状態を確認して」

➔ browser_snapshot（軽量・高速・正確）

✘ こうしない

「スクリーンショットを撮って」（毎回）

➔ 画像は最終確認やレイアウト確認時だけに限定しましょう。
