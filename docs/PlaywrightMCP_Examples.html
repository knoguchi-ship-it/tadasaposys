実践例 & トラブルシューティング

タダサポ管理システムでの具体的なテストシナリオ集

読了目安: 約10分

1. チートシート（よく使うプロンプト5選）

# やりたいこと プロンプト
1 アプリを開いて確認 「Playwright MCP で http://localhost:3000 を開いて、画面の状態を確認して」
2 全機能テスト 「テスト動作して、エラーが出ないか確認してください」
3 モバイル表示確認 「画面を375x812にしてモバイル表示を確認して」
4 エラーチェック 「コンソールエラーがないか確認して」
5 ブラウザ終了 「ブラウザを閉じて」

2. 事前準備

テスト前にローカルサーバーを起動してください 
npx serve -s . -l 3000

→ http://localhost:3000 でアプリが表示されます。

3. テストシナリオ

以下の5つのシナリオをコピペして Claude Code に貼り付けるだけでテストできます。

シナリオ A: 回数超過の動作確認

年間利用10回に達した事業所では、担当するボタンが非表示になり、代わりに回数超過ボタンが表示されます
以下の手順でテストして：

1. Playwright MCP で http://localhost:3000 を開いて
2. 未対応タブで「リミット介護サービス」を選択
3. 今年度利用数が 10/10 で赤く表示されているか確認
4. 下部に「回数超過」ボタンが表示されているか確認
5. 「担当する」ボタンが非表示であることを確認

シナリオ B: 過去記録の表示確認

2回目以降の対応中案件では、過去の対応記録セクションが表示されます
以下をテストして：

1. 対応中タブで案件を1つ選択
2. 「過去の対応記録」セクションが存在するか確認
3. 過去の記録を展開して内容が表示されるか確認

シナリオ C: 検索機能の確認

キーワード検索と期間検索で案件を絞り込めます
検索機能をテストして：

1. 検索アイコンをクリックして検索パネルを開く
2. キーワード欄に「訪問」と入力
3. フィルタされた結果が正しいか確認
4. クリアボタンで検索をリセット

シナリオ D: レスポンシブ確認

モバイル幅では案件一覧が全幅表示になり、案件選択で詳細画面に切り替わります
モバイル表示を確認して：

1. 画面を 375x812 にリサイズ
2. 案件一覧が全幅で表示されるか確認
3. 案件をクリックして詳細画面に遷移するか確認
4. 「一覧に戻る」ボタンが表示されるか確認

シナリオ E: 完了案件の再開制限確認

3/3回の案件では再開ボタンが非表示。年間10回上限の事業所でも再開不可
再開制限をテストして：

1. 完了タブで「いとう在宅ケア」（3/3回）を選択
2. 再開ボタンが非表示であることを確認
3. 完了タブで別の案件（1/3回）を選択
4. 再開ボタンが表示されていることを確認
5. コンソールエラーがないことも確認して

4. トラブルシューティング Q&A

よくある問題とその解決策をまとめました。

Q: /mcp で playwright が表示されない

  1. プロジェクトルート直下に .mcp.json が存在するか確認
  2. .mcp.json が正しいJSON形式か確認（末尾カンマに注意）
  3. Claude Code を完全に終了して再起動
  4. ターミナルで npx -y @playwright/mcp@0.0.68 --help が動くか確認
  5. ファイアウォールやプロキシがnpxをブロックしていないか確認

Q: ブラウザが起動しない

  1. ファイアウォールが Chromium をブロックしていないか確認
  2. npx playwright install chromium で手動インストール
  3. --headless を追加してヘッドレスモードで試す
  4. --isolated で一時プロファイルで試す

Q: タイムアウトエラーが出る

  • ローカルサーバーが起動しているか確認: npx serve -s . -l 3000

Q: 操作対象の要素が見つからない

  • browser_snapshot で現在のアクセシビリティツリーを取得してから操作を指示する。「まず画面の状態を確認して」と伝える

Q: コンテキスト（トークン）を消費しすぎる

  • スクリーンショットは最終確認時のみに限定
  • browser_snapshot を主に使用
  • 不要なタブは閉じる
  • テスト終了後は browser_close で閉じる

Q: セッションが残ってしまう

npx @playwright/mcp@0.0.68 close-all

5. よくあるミス集

Do
ローカルサーバーを起動してからテスト
Don't
サーバー起動を忘れてタイムアウト
Do
1つずつ順番にテスト
Don't
全テストを一度に指示して混乱
Do
操作前にスナップショットで状態確認
Don't
前回のスナップショットのref番号で操作（ページが変わるとref番号も変わる）
