ツールリファレンス

Claude が使用する全ツールの一覧と使い方

リファレンス
ⓘ
ユーザーがツール名を覚える必要はありません。自然な日本語で指示すれば、Claude が適切なツールを自動的に選択します。このページは「こういう操作ができる」という参考としてご活用ください。

1 ナビゲーション（3ツール）

ページを開いたり、移動したり、待機したりする基本操作

ツール名機能指示例
browser_navigateURLを開く「〇〇を開いて」
browser_navigate_backブラウザの戻る「前のページに戻って」
browser_wait_forテキスト表示/非表示の待機、指定秒数待機「3秒待って」「ローディングが消えるまで待って」

2 画面取得（2ツール） 核心機能

ページの状態を把握するための最も重要なツール

ツール名機能指示例
browser_snapshotアクセシビリティツリー取得（推奨：構造化テキスト。操作にはこちらを使用）「画面の状態を確認して」
browser_take_screenshotスクリーンショット撮影（PNG/JPEG）（視覚確認時のみ。操作には使えない）「スクショを撮って」

3 操作（8ツール）

ページ上の要素をクリック、入力、選択する

ツール名機能指示例
browser_click要素をクリック（左/右/中ボタン、ダブルクリック対応）「〇〇ボタンを押して」
browser_typeテキスト入力（submitオプションでEnter送信可）「検索欄に〇〇と入力して」
browser_fill_form複数フォームフィールドを一括入力「フォームに名前とメールを入力して」
browser_select_optionドロップダウン選択（複数選択対応）「方法をGoogleMeetに変えて」
browser_hoverマウスホバー「メールアドレスにカーソルを合わせて」
browser_dragドラッグ&ドロップ「要素をドラッグして」
browser_press_keyキーボード入力（ArrowLeft, Enter, a等）「Escキーを押して」
browser_file_uploadファイルアップロード（複数ファイル対応）「ファイルをアップロードして」

4 タブ管理（1ツール）

ブラウザタブの操作

ツール名機能指示例
browser_tabsタブの一覧表示・新規作成・閉じる・切り替え（action: list/new/close/select）「開いているタブを見せて」

5 デバッグ＆情報取得（3ツール）

エラー確認やネットワーク監視に使う

ツール名機能指示例
browser_console_messagesコンソールログ取得（error/warning/info/debugレベル指定可）「コンソールエラーを確認して」
browser_network_requestsネットワークリクエスト一覧「APIリクエストを確認して」
browser_evaluateページ上で任意のJavaScriptを実行「DOMの状態をJSで確認して」

6 ユーティリティ（5ツール）

画面サイズ変更、ダイアログ応答などの補助機能

ツール名機能指示例
browser_resizeブラウザウィンドウのサイズ変更「画面を375x812にして」
browser_handle_dialogalert/confirm/promptダイアログの応答「確認ダイアログでOKを押して」
browser_installブラウザの手動インストール「ブラウザをインストールして」
browser_run_codePlaywrightコードスニペットを直接実行「Playwrightコードで〇〇して」
browser_closeブラウザを閉じる「ブラウザを閉じて」

+ 追加 Capability ツール

ⓘ
以下のツールはデフォルトでは無効です。使用するには .mcp.json の args に --caps オプションを追加してください。

Vision（--caps core,vision で有効化）

座標ベースのマウス操作。スクリーンショット上の位置を指定して操作する場合に使用

ツール名機能
browser_mouse_click_xy指定座標でクリック
browser_mouse_move_xy指定座標にマウス移動
browser_mouse_drag_xy指定座標へドラッグ
browser_mouse_downマウスボタン押下
browser_mouse_upマウスボタン解放
browser_mouse_wheelマウスホイールスクロール

PDF（--caps core,pdf で有効化）

ツール名機能指示例
browser_pdf_saveページをPDFとして保存「このページをPDFで保存して」

DevTools（--caps core,devtools で有効化）

テスト用のアサーション・ロケーター生成ツール

ツール名機能
browser_generate_locatorテスト用ロケーターの生成
browser_verify_element_visible要素の表示を検証
browser_verify_list_visibleリストの表示を検証
browser_verify_text_visibleテキストの存在を検証
browser_verify_value要素の値を検証

⚙ Capability有効化の方法

.mcp.json の args に --caps オプションを追加して、有効にしたい機能をカンマ区切りで指定します。

{
  "mcpServers": {
    "playwright": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@playwright/mcp@0.0.68", "--caps", "core,vision,pdf,devtools"]
    }
  }
}
ⓘ
本プロジェクトの設定: 現在の .mcp.json では --caps を指定していないため、デフォルトの core（計22ツール）が有効です。
